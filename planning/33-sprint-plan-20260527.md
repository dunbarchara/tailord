## Day 2 — User Identity & Data Model Refactor

Split the monolithic `users` table into four concern-separated tables. Removes null-bloat (5 Notion columns null for most users), decouples Google as the only auth provider, adds `updated_at`/`deleted_at` for audit and tombstone support.

### Migration

- [x] Wrote Alembic migration `a7b8c9d0e1f2` — single atomic migration (create + backfill + drop); runs at `alembic upgrade head` on startup, so two-migration split would not work with current deployment model
- [x] Creates `auth_identities` table (provider-neutral OAuth subjects; UNIQUE `(provider, subject)`)
- [x] Creates `user_profiles` table (1:1 with users; `username_slug` UNIQUE indexed; `communication_email` for future digest)
- [x] Creates `user_integrations` table (UNIQUE `(user_id, provider)`; JSONB `credentials` + `provider_metadata`)
- [x] Adds `updated_at`, `deleted_at` to `users`
- [x] Backfills `auth_identities` from `users.google_sub` (provider="google")
- [x] Backfills `user_profiles` from users profile fields (1 row per user)
- [x] Backfills `user_integrations` from users Notion fields (only where `notion_access_token IS NOT NULL`)
- [x] Drops all migrated columns from `users`: `google_sub`, profile fields (6), Notion fields (5) — 239 tests pass post-migration

### Backend models (`database.py`)

- [x] `User` stripped to: `id`, `email` (nullable), `name`, `status`, `is_admin`, `created_at`, `updated_at`, `deleted_at`
- [x] Added `AuthIdentity` model — `provider`, `subject`, `email`, `connected_at`, `updated_at`; back_populates User
- [x] Added `UserProfile` model — all profile fields + `communication_email`; back_populates User
- [x] Added `UserIntegration` model — `provider`, `credentials` (JSONB), `provider_metadata` (JSONB, DB col name `metadata`); back_populates User
- [x] `User.profile` relationship: `lazy="selectin"` (always co-loaded)
- [x] `User.integrations` relationship: `lazy="selectin"` (always co-loaded)
- [x] `User.candidate_name` property: delegates to `user.profile.preferred_first/last` → `user.name` → `user.email`

### Backend auth (`deps_user.py`)

- [x] `get_current_user`: looks up `AuthIdentity(provider="google", subject=x_user_id)` instead of `User.google_sub`
- [x] New user path: creates `User` → `UserProfile` (with avatar_url) → `AuthIdentity` atomically; double-IntegrityError catch for race conditions
