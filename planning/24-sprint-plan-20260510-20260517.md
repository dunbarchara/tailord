## Day 21 — Editable Tailorings Workbench

### Theme
The Analysis view is the user's workbench. The platform does its best to parse job postings
correctly but cannot guarantee perfect output. The tailoring is ultimately the user's advocacy
artifact, so we surface controls to correct mistakes and express preferences.

### Toolbar Design Decision
Context-aware controls per view — only render controls applicable to the selected tab, rather
than always showing the full set greyed out.

Rationale:
- Analysis is a workbench — edit controls belong here
- Posting is a presentation layer derived from Analysis — no editing needed; greyed-out controls
  would imply editability where there is none
- Letter is a distinct writing artifact — it deserves its own controls (rephrase, tone) that share
  no overlap with Analysis controls
- Cognitive load: three distinct toolbars are clearer than one toolbar with conditional greying
