---
name: archive-planning
description: Periodic spring cleaning for planning/. Extracts all open/deferred work items from sprint plans, writes a structured roadmap doc, then archives all first-level .md files into a dated directory.
---

Archive the `planning/` directory and produce a fresh remaining-work summary.

## Step 1 — Get today's date

```bash
date +%Y%m%d
```

Store the result as `YYYYMMDD` (e.g. `20260524`). Use this value in the archive directory name and in the preamble of both output files.

---

## Step 2 — Find work-item files and read existing dump

1. List all `.md` files **directly in `planning/`** (not in subdirectories, not inside any `archive_*/` directory).
2. Use Grep to find which of those files contain `- [ ]` or `- [~]` markers. Read every file that matches — these are the sprint plans and task docs that have open or deferred work.
3. **Also read `planning/01-remaining-work-dump.md`** if it exists (or any file matching `planning/*remaining-work-dump*`). This is the prior dump from the last archive run.
   - Cross-reference its `- [ ]` and `- [~]` items against `- [x]` lines in the sprint plans.
   - Items from the old dump that appear completed (marked `[x]`) in any sprint plan are **dropped** — do not carry them forward.
   - Items from the old dump that are still open (not marked `[x]` anywhere) **carry forward** into the new dump under a clearly labelled section.

---

## Step 3 — Write `planning/remaining-work-dump.md`

Extract every `- [ ]` (pending) and `- [~]` (deferred) line from:
- All sprint plan files scanned in Step 2
- The old `*remaining-work-dump*` file (for carried-forward items only)

For each item, preserve enough surrounding context (section header, day label, epic name) to understand what it refers to. Group by **source file**. This file is a flat dumping ground — no thematic reorganisation yet.

Format:
```markdown
# Remaining Work — Raw Dump YYYY-MM-DD

*Generated by /archive-planning. Source files listed below.*

---

## From: 24-sprint-plan-20260510-20260517.md
- [ ] Item A (from "## Day 22 — ...")
- [~] Item B — deferred: reason (from "## Day 23 — ...")

## From: 02-sprint-20260410-20260417.md
- [ ] Item C (from "### Feature 1.1 — ...")

## From: 01-remaining-work-dump.md (carried forward — not yet completed)
- [ ] Item D
```

If `planning/remaining-work-dump.md` already exists, overwrite it.

---

## Step 4 — Write `planning/remaining-work-formatted.md`

Read the dump file produced in Step 3, then reorganize all items into an **Epic → Feature → Task** hierarchy. Do not preserve source-file grouping — cluster thematically using judgment. Suggested epics (adapt as needed):

- LLM Quality & Evaluation
- Experience Capture
- Tailoring & Generation
- Frontend UX
- Platform Hardening
- Infra & Observability
- Developer Experience

Preserve deferred status as `[~]`, pending as `[ ]`. Add a preamble noting the archive date and the source files consumed.

Format (matches `planning/02-sprint-20260410-20260417.md`):
```markdown
# Remaining Work — Reorganized YYYY-MM-DD

*Aggregated from: [list source files]. Prior planning archived to `planning/archive_YYYYMMDD/`.*

---

## Epic 1 — LLM Quality & Evaluation
**Vision:** One sentence on why this epic matters.

### Feature 1.1 — ...
- [ ] Task
- [~] Deferred task — reason

## Epic 2 — ...
```

If `planning/remaining-work-formatted.md` already exists, overwrite it.

---

## Step 5 — Archive everything else

1. Create directory `planning/archive_YYYYMMDD/`.
2. Move every first-level `.md` file in `planning/` to that directory — **except** `remaining-work-dump.md` and `remaining-work-formatted.md`.
3. Do **not** move subdirectories (`experience_capture/`, any existing `archive_*/`, etc.).
4. Use `mv` via Bash for each file.

```bash
mkdir -p planning/archive_YYYYMMDD
mv planning/<file>.md planning/archive_YYYYMMDD/
# repeat for each file
```

---

## Verification checklist

After completing all steps, confirm:

- [ ] `planning/remaining-work-dump.md` exists and contains `- [ ]` / `- [~]` items grouped by source file
- [ ] `planning/remaining-work-formatted.md` exists and organises those items by epic
- [ ] `planning/archive_YYYYMMDD/` exists and contains all the old `.md` files
- [ ] `planning/` first level now only has `remaining-work-dump.md`, `remaining-work-formatted.md`, and subdirectories (no other `.md` files)
